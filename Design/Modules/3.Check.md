# Module 3 - Check

## Purpose

Check helps users catch report and submission problems before exporting.

## MVP Responsibilities

- Analyze Markdown structure.
- Detect common missing sections.
- Detect evidence gaps.
- Detect weak or risky content markers.
- Return actionable issues with severity and target location.

## MVP Checks

- Missing table of contents.
- Missing conclusion.
- Missing references.
- Missing member responsibility table.
- Missing GitHub/demo/deploy links when using a software project template.
- Image path is empty or likely broken.
- Heading jumps levels.
- Figure/table caption missing.
- Table is likely too wide.
- Code block has no language.
- Text contains `TODO`, `fix later`, or `lorem ipsum`.

## Issue Shape

```ts
type ReportIssue = {
  id: string;
  severity: "error" | "warning" | "info";
  module: "write" | "format" | "check" | "export";
  message: string;
  suggestion: string;
  sectionId?: string;
  line?: number;
};
```

## Acceptance Criteria

- Checker runs without network access.
- Issues are grouped by severity.
- Each issue includes a short fix suggestion.
- User can re-run checker after edits and see updated results.
