# Module 2 — Format

> **Vị trí trong stack:** `src/modules/format/` (TechnicalStack §9).
> **Tuần xây dựng:** W3 (heading parser + numbering + basic TOC) → W7 (caption hardening, list of figures/tables, page-break) — theo `Design/RoadMap/MasterRoadMap.md`.
> **Canonical types:** `ReportProject`, `ReportSection` (Contract W1).
> **Input contract:** Format đọc `ReportProjectBundle` và trả `FormattedReport`. Format là nơi duy nhất sinh heading/caption numbering.

---

## 1. Purpose

Format biến Markdown thô thành báo cáo **trông như đã được căn chỉnh thủ công trong word processor**. Đây là "lớp học thuật": đánh số chương mục (`1`, `1.1`, `1.1.1`), sinh mục lục (TOC), đánh số hình/bảng, áp preset A4 + typography học thuật, và chuẩn bị nội dung cho PDF/DOCX.

Vì sao giữ chân người dùng: sinh viên sợ nhất khâu **format đúng chuẩn nộp bài** (font, lề, số trang, mục lục tự động). Format làm việc đó **deterministic** — cùng Markdown + metadata luôn ra cùng numbering, nên preview = PDF = DOCX. Người dùng không phải mở Word để căn tay nữa.

Nguyên tắc xương sống: Format **không** render thẳng ra file. Nó là một tập **transform trên AST** (mdast/hast) dùng chung pipeline với Check & Export (TechnicalStack §3). Format chuẩn hoá, Export render.

---

## 2. Scope

### In scope (MVP — W3, hardening W7)

- Normalize heading numbering: `1`, `1.1`, `1.1.1` từ cấu trúc heading.
- Generate table of contents data (`TocNode[]`) từ heading đã đánh số.
- Đánh số + chuẩn hoá caption hình (Figure) và bảng (Table).
- A4 layout presets (margin, font, size, line-height, alignment).
- Academic typography presets (Times New Roman 13/14, line-height 1.5, justify body).
- Chapter behavior: chương lớn bắt đầu trang mới trong PDF/DOCX.
- Header / footer / page number config (data, render do Export).
- (W7) List of figures / list of tables.
- Cross-reference hình/bảng theo số sinh tự động.
- `FormatSettings.includeToc` quyết định có sinh TOC hay không; user không cần viết heading "Mục lục" trong Markdown.

### Out of scope (theo `ProductPRD.md` §6 Non-goals)

- **No manual pixel-dragging layout** — không phải trình bày tự do kiểu Word; chỉ preset deterministic từ Markdown.
- **No rendering to file** — Format chỉ transform AST + sinh metadata layout; PDF/DOCX là việc của Module 4.
- **No "convert mọi định dạng"** — chỉ format báo cáo Markdown của chính người dùng.
- **No bibliography manager / citation engine** (Zotero-style) ở MVP — references chỉ là section + rule kiểm tra (Module 3), không tự resolve citation.
- **No AI auto-formatting** ở Phase 1.

## 3. Data Model

### 3.1 Canonical types (giữ NGUYÊN — Contract W1)

`ReportProject`, `ReportSection` — xem `Design/Modules/1.Write.md` §3.1. Format đọc `ReportProjectBundle.project.sections[]`, `bundle.project.metadata`, `bundle.assets`, và `bundle.formatSettings`, **không** đổi shape canonical.

### 3.2 Module-specific types

```ts
// src/modules/format/types.ts

// Node mục lục sinh từ heading đã đánh số.
type TocNode = {
  id: string;            // anchor slug, vd "1-1-kien-truc"
  number: string;        // "1.1"
  text: string;          // "Kiến trúc hệ thống"
  level: number;         // 1..6 (h1..h6)
  sectionId: string;     // ReportSection chứa heading này
  children: TocNode[];   // cây phân cấp
};

// Caption hình/bảng đã đánh số.
type CaptionEntry = {
  id: string;            // anchor để cross-reference
  kind: "figure" | "table";
  number: number;        // 1, 2, 3 (chạy toàn báo cáo hoặc theo chương — xem config)
  label: string;         // "Hình 1", "Bảng 2"
  text: string;          // mô tả caption
  sectionId: string;
};

// Preset layout học thuật áp khi render.
type FormatPreset = {
  id: string;            // "academic-default" | "tnr-13" | ...
  page: "A4";            // MVP chỉ A4
  margin: { top: string; right: string; bottom: string; left: string }; // cm/mm
  fontFamily: string;    // "Times New Roman"
  fontSizePt: 13 | 14;
  lineHeight: number;    // 1.5
  bodyAlign: "justify" | "left";
  chapterStartsNewPage: boolean; // true → page-break trước h1/chương lớn
  captionNumbering: "continuous" | "per-chapter"; // "Hình 1" vs "Hình 1.1"
  header?: string;       // template chuỗi, vd "{title}"
  footer?: string;       // vd "Trang {page} / {pages}"
};

type FormatSettings = {
  presetId: string;
  includeToc: boolean;
  includeListOfFigures: boolean;
  includeListOfTables: boolean;
  captionNumbering: "continuous" | "per-chapter";
};

// Kết quả tổng hợp của Format — Export tiêu thụ cái này.
type FormattedReport = {
  toc: TocNode[];
  figures: CaptionEntry[];
  tables: CaptionEntry[];
  preset: FormatPreset;
  // AST đã transform (heading đánh số, caption đánh số, anchor cross-ref)
  // giữ ở dạng hast để Export render HTML/PDF, và mdast cho DOCX.
};
```

### 3.3 Caption and cross-reference syntax

MVP khóa syntax đơn giản, dễ parse:

```md
![Màn hình đăng nhập](asset:login)
```

- `asset:login` trỏ tới `ReportAsset.id`.
- Alt text (`Màn hình đăng nhập`) là caption mặc định.
- Figure id mặc định sinh từ asset id: `fig:login`.
- Cross-reference `[#fig:login]` là later/hardening; MVP chỉ cần giữ id đủ ổn định để không phải đổi syntax sau.

Table caption/cross-reference nâng cao để Phase sau. MVP ưu tiên figure captions qua image alt text, tránh nhiều false positive do đoán caption từ dòng tự do.

## 5. Processing Logic

### 5.1 Heading numbering algorithm

1. Gộp `sections` theo `order` → một mdast tổng (hoặc duyệt từng section nối tiếp giữ counter chung).
2. Duyệt tuần tự mọi `heading` node, lấy `depth` (1..6).
3. Giữ mảng counter `counters[1..6]`. Khi gặp heading depth `d`:
   - `counters[d] += 1`.
   - Reset mọi `counters[k]` với `k > d` về 0.
   - Number = join các `counters[1..d]` khác 0 bằng `.` → vd `1.2.1`.
4. Gán `number` vào node, sinh anchor slug từ text (`slugify`) → `id` cho cross-reference & TOC.
5. **Heading-jump handling:** nếu depth nhảy cấp (vd h1 → h3 bỏ h2), vẫn đánh số theo cấp thực nhưng **đánh dấu** để Module 3 (rule `heading-jump`) báo warning. Format KHÔNG tự sửa cấu trúc của người dùng.

### 5.2 Table of contents generation

1. Sau numbering, lọc heading có depth ≤ ngưỡng TOC (mặc định ≤ 3).
2. Tạo `TocNode` phẳng `{ number, text, level, id, sectionId }`.
3. Build cây: dùng stack theo `level` để nest `children` (level lớn hơn → con của level nhỏ hơn gần nhất).
4. Output `TocNode[]` (cây) cho TOC preview + Export.

### 5.3 Caption numbering algorithm

1. Duyệt AST tìm:
   - **Figure:** `image` node có `url` dạng `asset:<id>` và alt text làm caption.
   - **Table:** GFM `table` node. MVP chưa bắt buộc table caption syntax nâng cao; Check có thể cảnh báo nếu template yêu cầu caption.
2. Counter riêng cho figure và table.
   - `continuous`: số chạy 1,2,3… toàn báo cáo.
   - `per-chapter`: reset theo chương → `1.1`, `1.2`, `2.1` (chương lấy từ `counters[1]`).
3. Sinh `label` ("Hình N" / "Bảng N"), gán `id` anchor. Với figure asset, id mặc định là `fig:<assetId>`.
4. Chèn caption đã đánh số ngay dưới hình / trên bảng (theo chuẩn học thuật).
5. Cross-reference `[#fig:abc]` là later/hardening; nếu có trong Markdown mà không resolve được, Check cảnh báo nhưng Format không crash.

### 5.3b Double-numbering prevention

- Format **không** thêm số vào heading đã có số hard-coded theo kiểu `1.`, `1.1`, `Chương 1`.
- Thay vào đó, Format gắn flag để Check báo `hardcoded-heading-number`.
- Template chuẩn phải viết `# Mở đầu`, không viết `# 1. Mở đầu`.
- Output chỉ có một nguồn numbering: Format counters.

### 5.4 Layout preset application

1. Map `FormatPreset` → print CSS (cho HTML/PDF) và → cấu hình `docx` (cho DOCX, do Module 4 dùng).
2. `chapterStartsNewPage: true` → chèn marker page-break trước mỗi h1/chương lớn (CSS `break-before: page` cho PDF; `PageBreak` cho DOCX).
3. Header/footer template resolve biến: `{title}` ← `project.title`, `{page}`/`{pages}` ← do Export target điền lúc render.
4. Output `FormattedReport` để Export tiêu thụ — **một nguồn cho cả 3 target**.

## 6. Edge Cases & Smart-Operation Notes ("gia vị vận hành")

- **Empty headings:** Heading rỗng (`#` không text) → bỏ qua khỏi TOC, không sinh anchor rỗng; Module 3 có thể flag.
- **Heading jump (h1→h3):** Vẫn đánh số được nhưng đánh dấu warning; Format không tự "vá" cấp để tránh sửa sai ý người dùng.
- **Duplicate heading text:** Hai heading cùng text → slug trùng → append `-2`, `-3` để anchor luôn unique (TOC/cross-ref không lẫn).
- **Caption thiếu:** Hình/bảng không có caption → vẫn đánh số nhưng để text rỗng; Module 3 rule `missing-caption` báo warning.
- **Huge table (quá rộng):** Bảng nhiều cột → Format gắn cờ "wide-table" để preview/PDF cho `font-size` nhỏ hơn hoặc xoay landscape khối đó; Module 3 cảnh báo `table-too-wide`.
- **Nested deep headings (h5/h6):** Cho phép đánh số tới `1.1.1.1.1.1`; TOC mặc định cắt ở depth 3 để mục lục không rối.
- **Page-break giữa caption và hình:** Đặt rule giữ hình + caption cùng trang (CSS `break-inside: avoid`) để PDF không tách caption khỏi ảnh.
- **Chapter-break khi chỉ có 1 chương:** `chapterStartsNewPage` vẫn áp nhưng không tạo trang trắng thừa đầu báo cáo.
- **TOC không nằm trong Markdown:** `includeToc=false` thì không sinh TOC; `includeToc=true` thì Export chèn TOC từ `FormattedReport.toc`. Không tìm heading "Mục lục" trong nội dung người dùng.
- **TOC số trang:** HTML/preview không có số trang thực; số trang TOC chỉ điền chính xác ở PDF target có layout trang — preview hiện "…" hoặc ẩn cột số trang.
- **Mermaid/figure đánh số:** Mermaid diagram tính như "Hình" nếu có caption; nếu không thì không đánh số, tránh đếm nhầm.
- **Cross-reference gãy:** `[#fig:xyz]` trỏ tới id không tồn tại → render literal + Module 3 cảnh báo (không crash).

## 7. Dependencies

### Modules

- **Module 1 (Write):** Cung cấp `sections[].markdown` + `metadata` (title cho header/cover).
- **Module 3 (Check):** Đọc các cờ Format gắn (heading-jump, wide-table, missing-caption) để báo issue; cùng AST.
- **Module 4 (Export):** Tiêu thụ `FormattedReport` (AST đã transform + preset) để render HTML/PDF/DOCX — đảm bảo numbering đồng nhất 3 format.

### Locked stack libs (TechnicalStack.md)

- `unified` + `remark-parse` + `remark-gfm` + `remark-rehype` — parse & transform AST.
- `remark-math` + `rehype-katex` + `katex` — giữ math khi format.
- `rehype-highlight` — code blocks trong nội dung format.
- Print CSS nhúng — áp `FormatPreset` cho HTML/PDF (render bởi Module 4).
- `zod` — validate `FormatPreset`.
- `Vitest` — test numbering / TOC / caption algorithm (deterministic, regression-safe).

## 9. Acceptance Criteria

- Exported HTML preview dùng cùng heading/caption numbering như PDF/DOCX.
- Figures và tables tham chiếu được bằng số sinh tự động (cross-reference).
- Formatting logic **deterministic** từ Markdown + metadata (cùng input → cùng output).
- TOC (`TocNode[]`) phản ánh đúng cây heading sau khi đánh số.
- `FormatPreset` (A4, TNR 13/14, line-height 1.5, justify, chapter-break) áp được và map sang cả print CSS lẫn cấu hình DOCX.
- Heading-jump / wide-table / missing-caption được gắn cờ cho Module 3, Format không tự sửa cấu trúc người dùng.
- Header/footer/page-number config sinh đúng data để Export điền.
