# Module 5 — Present

> **Vị trí trong stack:** `src/modules/present/` (sẽ tạo ở Phase 3 — TechnicalStack §9 chưa liệt vì là later).
> **Tuần xây dựng:** W9 (slide outline + timeline + speaker assign) → W10 (script + Q&A) — Phase 3, `Design/RoadMap/MasterRoadMap.md`.
> **Canonical types:** `ReportProject`, `ReportSection` (Contract W1) — Present đọc, không đổi shape.
> **STATUS:** **NOT in MVP.** Tài liệu này định hướng sản phẩm; mọi quyết định kỹ thuật vẫn phải xin approve trước khi cài (TechnicalStack §8 — `pptxgenjs` deferred).

---

## 1. Purpose

Present biến một báo cáo hoàn chỉnh thành **vật liệu hỗ trợ thuyết trình**: outline slide, kịch bản nói (speaker script), bộ câu hỏi phản biện (defense Q&A), phân công người nói theo section, và checklist nộp bài cuối cùng.

Vì sao giữ chân người dùng (định hướng): sau khi viết + format + export báo cáo, bước tiếp theo của sinh viên/nhóm phần mềm luôn là **bảo vệ / thuyết trình**. Nếu ReportSupporter đi tiếp được tới đây, nó phủ trọn vòng đời "viết báo cáo → nộp → bảo vệ" thay vì dừng ở file PDF. Đây là phần mở rộng giữ người dùng ở lại cho lần sau.

> ⚠️ Module này **không thuộc MVP đầu tiên** (đúng tinh thần stub gốc), nhưng được cấu trúc sẵn để Phase 3 không phải thiết kế lại từ đầu. Mọi thứ dưới đây là **forward-looking**.

---

## 2. Scope

### In scope (Phase 3 — W9/W10)

- Sinh slide outline từ `ReportSection[]` (giữ đúng dòng chương mục báo cáo).
- Sinh speaker script (kịch bản nói) tham chiếu screenshot/evidence.
- Sinh defense Q&A (câu hỏi phản biện khả dĩ cho báo cáo đồ án).
- Phân công người nói theo section (speaker assignment).
- Presentation timeline (phân bổ thời gian từng phần).
- Export slides PPTX (`pptxgenjs`) + PDF slides.
- Final submission checklist.

### Out of scope (theo `ProductPRD.md` §6 Non-goals + giữ kỷ luật Phase)

- **No MVP delivery** — không xây ở Phase 1/2; chỉ Phase 3.
- **No AI content generation ở Phase 1** — sinh outline/script ban đầu là **deterministic từ cấu trúc báo cáo**; phần "AI-assisted" chỉ bật ở W11 sau **explicit user action** (RoadMap W11), người dùng giữ quyền kiểm soát nội dung.
- **No realtime co-presenting / live audience** — không streaming, không realtime (Non-goal).
- **No cloud / no login** — vẫn giữ posture local-first; PPTX sinh client rồi tải về.
- **No full slide design editor** (kiểu Canva/PowerPoint WYSIWYG) — chỉ outline + nội dung; không trình bày pixel tự do.
- **No new library ngoài stack** — `pptxgenjs` đã được khoá trước cho Phase 3 nhưng **vẫn phải xin approve** khi cài (TechnicalStack §8).

## 3. Data Model

### 3.1 Canonical types (giữ NGUYÊN — Contract W1)

### 3.2 Module-specific types (dự kiến — Phase 3)

```ts
// src/modules/present/types.ts (forward-looking)

// Outline slide sinh từ section báo cáo.
type SlideOutline = {
  id: string;
  fromSectionId: string;     // map về ReportSection nguồn
  order: number;
  title: string;             // tiêu đề slide (thường = heading section)
  bullets: string[];         // ý chính rút từ nội dung section
  speakerId?: string;        // người nói được gán
  evidenceRefs: string[];    // id evidence/screenshot tham chiếu
  estimatedSeconds?: number; // thời lượng dự kiến (timeline)
};

// Kịch bản nói cho 1 slide.
type SpeakerScript = {
  slideId: string;
  speakerId?: string;
  script: string;            // đoạn nói deterministic từ nội dung section
  cues: string[];            // gợi ý "chỉ vào hình X", "demo link Y"
};

// Câu hỏi phản biện khả dĩ.
type DefenseQA = {
  id: string;
  question: string;
  suggestedAnswer: string;
  relatedSectionId?: string;
  topic: "scope" | "tech" | "result" | "limitation" | "future"; // nhóm câu hỏi
};

// Phân công người nói.
type Speaker = {
  id: string;
  name: string;              // lấy từ project.metadata.members nếu có
  assignedSlideIds: string[];
};

// Timeline thuyết trình.
type PresentationTimeline = {
  totalSeconds: number;
  slots: { slideId: string; speakerId?: string; seconds: number }[];
};

// Gói trình bày tổng hợp + target export.
type PresentTarget = "pptx" | "pdf-slides";

type PresentPackage = {
  outline: SlideOutline[];
  scripts: SpeakerScript[];
  qa: DefenseQA[];
  speakers: Speaker[];
  timeline: PresentationTimeline;
  submissionChecklist: { id: string; label: string; done: boolean }[];
};
```

## 5. Processing Logic (dự kiến — Phase 3)

### 5.1 Pipeline: Generate slide outline (deterministic)

1. Đọc `ReportSection[]` theo `order` + heading đã đánh số (từ Module 2).
2. Mỗi chương lớn (h1) / mục chính → một (hoặc vài) `SlideOutline`; title = heading.
3. Rút `bullets` từ nội dung: lấy câu chủ đề / list item đầu mục (deterministic, không AI ở Phase 3 ban đầu).
4. Map `evidenceRefs` từ image/link nodes trong section (tái dùng Evidence Kit — W5).
5. Giữ **đúng dòng chương mục** của báo cáo (acceptance gốc).

### 5.2 Pipeline: Speaker assignment + timeline

1. Lấy `members` từ `project.metadata` → tạo `Speaker[]`.
2. Gán slide cho speaker (mặc định chia đều theo số chương / cho người dùng kéo-thả).
3. Tính `PresentationTimeline`: phân bổ `estimatedSeconds` theo độ dài section → tổng.

### 5.3 Pipeline: Speaker script + Q&A (deterministic baseline → AI optional W11)

1. **Baseline (W10, deterministic):** sinh `SpeakerScript` từ bullets + caption hình ("Ở phần này, nhìn vào Hình N…").
2. **Q&A:** sinh `DefenseQA` theo template topic (scope/tech/result/limitation/future) bám nội dung section (vd thấy "deploy" → hỏi về môi trường triển khai).
3. **AI-assisted (W11, optional, behind explicit action):** nếu người dùng bật, gợi ý rewrite script/Q&A — **người dùng giữ quyền kiểm soát**, không tự ghi đè.

### 5.4 Pipeline: Export slides

1. **PPTX:** map `SlideOutline[]` → `pptxgenjs` slides (title + bullets + ảnh evidence).
2. **PDF slides:** tái dùng Module 4 (render HTML slide → Puppeteer PDF khổ slide).
3. Giữ cover/branding tối thiểu từ `metadata`.

## 6. Edge Cases & Smart-Operation Notes ("gia vị vận hành") (dự kiến)

- **Báo cáo quá dài → quá nhiều slide:** Gộp mục con thành 1 slide / giới hạn bullets/slide để slide không quá tải.
- **Section rỗng:** Bỏ qua khỏi outline, không tạo slide trắng.
- **Thiếu members trong metadata:** Speaker assignment để trống / 1 người mặc định; không crash.
- **Evidence ref gãy:** Slide tham chiếu evidence đã xoá → bỏ ref + ghi chú, không vỡ slide.
- **Timeline tổng vượt thời lượng cho phép:** Cảnh báo "tổng X phút vượt giới hạn" để người dùng cắt bớt.
- **AI off (mặc định):** Mọi outline/script/Q&A vẫn sinh được **không cần mạng/AI** (deterministic) — giữ tinh thần local-first; AI chỉ là lớp tăng cường tuỳ chọn.
- **PPTX lib chưa cài:** Nếu Phase 3 chưa approve `pptxgenjs` → nút PPTX disabled + ghi rõ "cần bật Phase 3"; PDF slides (qua Module 4) có thể chạy trước.
- **Mermaid/math trên slide:** Tái dùng cùng pipeline render (Module 4) để diagram/math lên slide đúng như báo cáo.

## 7. Dependencies

### Modules

- **Module 1 (Write):** Nguồn `ReportProject` + `sections` + `metadata.members` (speaker).
- **Module 2 (Format):** Heading đánh số + caption (slide title + cue "Hình N").
- **Module 4 (Export):** Tái dùng render pipeline cho PDF slides (Puppeteer).
- **Evidence Kit (W5, Phase 2):** Nguồn `evidenceRefs` cho slide/script.

### Locked stack libs (TechnicalStack.md §8 — DEFERRED, cần approve)

- `pptxgenjs` — export `.pptx` (**Phase 3, xin approve trước khi cài**).
- `puppeteer` — PDF slides (tái dùng Module 4).
- `unified`/`remark`/`rehype` + `katex` + `mermaid` — render nội dung slide nhất quán báo cáo.
- `zod` — validate `PresentPackage`.
- (W11 optional) AI assistant — **không** thuộc stack hiện tại; nếu thêm phải approve và đặt sau explicit user action (Non-goal Phase 1).

## 9. Acceptance Criteria (Later Phase)

- Slide outline giữ **cùng chapter flow** như báo cáo.
- Generated script tham chiếu key screenshots & evidence.
- Q&A tập trung vào câu hỏi phản biện khả dĩ cho báo cáo đồ án.
- Outline/script/Q&A baseline sinh được **deterministic, offline** (không bắt buộc AI).
- Speaker assignment + timeline hoạt động từ `metadata.members`.
- AI-assisted (W11) chỉ bật sau explicit user action; người dùng giữ quyền kiểm soát nội dung (Non-goal Phase 1 tôn trọng).
- Export PPTX/PDF slides nhất quán nội dung với báo cáo (cùng render pipeline).
- Không thêm library ngoài stack mà chưa approve (TechnicalStack §8).
