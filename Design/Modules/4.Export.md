# Module 4 — Export

> **Vị trí trong stack:** `src/modules/export/` (TechnicalStack §9).
> **Tuần xây dựng:** W1 (HTML stub + PDF/DOCX service placeholder) → W4 (Export MVP: HTML + PDF + DOCX + error handling) — theo `Design/RoadMap/MasterRoadMap.md`.
> **Canonical types:** `ReportProject`, `ReportSection`, `ReportIssue` (Contract W1).
> **MVP PDF rule:** PDF first path là browser print / print CSS từ HTML đã format. Puppeteer là hardening sau hoặc worker/service riêng, không khóa vào Vercel Function trong MVP.

---

## 1. Purpose

Export tạo ra **file người dùng cần để nộp**: `report.html`, `report.pdf`, `report.docx`. Đây là điểm cuối của vòng đời báo cáo — và là lúc người dùng nhận giá trị hữu hình.

Vì sao giữ chân người dùng: người dùng đến với ReportSupporter vì cuối cùng họ cần **một file PDF/DOCX đúng chuẩn nộp**. Nếu export trung thực (đúng A4, đúng font, đúng numbering, đúng page-break) thì họ không cần mở Word để sửa lại — đó là lý do họ quay lại thay vì dùng converter rời.

Nguyên tắc xương sống: **một nguồn, ba target**. Cùng `ReportProjectBundle` + `FormattedReport` → render ra HTML/PDF/DOCX **nhất quán**. HTML là source of truth tự chứa; PDF MVP dùng chính HTML đó qua browser print / print CSS; DOCX sinh từ mdast để giữ bản editable tốt nhất.

---

## 2. Scope

### In scope (MVP — W4)

- Markdown → HTML (`report.html`).
- Markdown → PDF bằng browser print / print CSS từ HTML đã format.
- Markdown → DOCX (`report.docx`) basic editable qua `docx` (sinh từ mdast).
- README/report template → PDF/DOCX (cùng pipeline).
- Cover page từ `metadata` (title/school/course/lecturer/members).
- Giữ nhất quán nội dung 3 format (numbering, table, code, image, caption).
- Export status + error surfacing + recover (W4).

### Out of scope (theo `ProductPRD.md` §6 Non-goals + TechnicalStack §4)

- **No "convert mọi định dạng"** — chỉ Markdown của người dùng → HTML/PDF/DOCX. Không nhận `.docx`/`.pdf` người khác để convert ngược.
- **No PDF toolbox** (merge/split/sign/OCR) — Export không phải bộ công cụ PDF; chỉ render báo cáo.
- **No Pandoc / LibreOffice headless / `@react-pdf/renderer`** — CẤM (TechnicalStack §4). PDF MVP = browser print/print CSS; DOCX = `docx`.
- **No cloud export / no email delivery** — file sinh ở client/Node route rồi tải về; không upload cloud (Non-goal no cloud).
- **No PPTX** ở MVP — slides là Phase 3 (Module 5, `pptxgenjs` deferred).

### 3.2 Module-specific types

```ts
// src/modules/export/types.ts

type ExportTarget = "html" | "pdf" | "docx";

type ExportStatus = "idle" | "running" | "done" | "error";

// Trả về từ export service methods (lỗi visible & recoverable)
type ExportResult =
  | { ok: true; blob: Blob }
  | { ok: false; error: ExportError };

// Một lần export (theo dõi status để hiển thị ở UI + lưu lịch sử local).
type ExportJob = {
  id: string;
  target: ExportTarget;
  projectId: string;
  status: ExportStatus;
  startedAt: string;          // ISO 8601
  finishedAt?: string;
  fileName: string;           // "report.pdf"
  error?: ExportError;        // khi status="error"
};

type ExportError = {
  stage: "merge" | "parse" | "format" | "render-html" | "render-pdf" | "render-docx";
  message: string;            // human-readable
  recoverable: boolean;       // true → cho retry
};

// Cấu hình cover page lấy từ ReportProject.metadata.
type CoverPageData = {
  title: string;
  school?: string;
  course?: string;
  lecturer?: string;
  members?: string[];
  date?: string;
};

// Input gom sẵn cho exporter (1 nguồn cho 3 target).
type ExportInput = {
  bundle: ReportProjectBundle;
  cover: CoverPageData;
  // Từ Module 2: AST đã đánh số + preset + toc/figures/tables.
  formatted: FormattedReport;  // xem Design/Modules/2.Format.md §3.2
};
```

## 5. Processing Logic

### 5.1 Shared front pipeline (chung cho cả 3 target)

1. **Merge:** Gộp `project.metadata` + `sections[]` (theo `order`) thành tài liệu logic + dựng `CoverPageData`.
2. **Parse:** Markdown → mdast (remark-parse + remark-gfm + remark-math) — **intermediate document model** (TechnicalStack §3).
3. **Format:** Áp transform Module 2 → `FormattedReport` (heading đánh số, caption đánh số, anchor, `FormatPreset`). Đây là điểm đảm bảo 3 target khớp numbering.
4. Tách nhánh theo `ExportTarget` (§5.2 / §5.3 / §5.4).

### 5.2 HTML target (`report.html`)

1. mdast → hast: remark-rehype + rehype-katex + rehype-highlight.
2. hast → HTML: rehype-stringify.
3. Nhúng **print CSS** từ `FormatPreset` (A4, TNR 13/14, line-height 1.5, justify, margin, page-break, header/footer).
4. Mermaid: nhúng script render client-side (mở file HTML trong trình duyệt sẽ render diagram).
5. Chèn cover page + TOC (từ `formatted.toc`) lên đầu.
6. Output 1 file `report.html` self-contained (ảnh base64 inline → offline-safe).

### 5.3 PDF target (`report.pdf`) — Exporter Routes

We support two distinct code paths for PDF generation:

#### 1. `exportPdfViaBrowserPrint()` (Core MVP, default)
- **Execution:** Runs entirely client-side.
- **Process:** Opens the formatted HTML from §5.2 in a print preview context, triggering the browser's native `window.print()` / "Save as PDF" dialogue.
- **Styling:** Controlled by standard print CSS (A4, margins, fonts, line-height, page-break, chapter-break).
- **Limitations & Acceptance Limits:** Header, footer, and page numbers are limited to browser print capabilities. PDF MVP đạt tiêu chuẩn nộp bài trên báo cáo mẫu nhưng **không cam kết** đạt độ chính xác hay đồng bộ tuyệt đối 100% về vị trí header/footer/đánh số trang như khi xuất DOCX/Word.

#### 2. `renderPdfWithPuppeteer()` (Later hardening only, default OFF)
- **Execution:** Runs server-side in a Node API route, disabled by default (dependencies are not installed in Core MVP).
- **Process:** Feeds the formatted HTML to a headless Chromium instance managed via a Puppeteer queue.
- **Advantages:** Allows pixel-perfect page numbering, headers, and footers.
- **Operation:** Handled behind a feature flag to protect server memory and function execution limits.

### 5.4 DOCX target (`report.docx`)

1. Duyệt **mdast** (không qua HTML) → map node → phần tử `docx`:
   - `heading` → `Paragraph` với `HeadingLevel` + số đã đánh.
   - `paragraph`/`text`/`strong`/`emphasis` → `TextRun` với style.
   - `table` (GFM) → `Table`/`TableRow`/`TableCell`.
   - `code` → `Paragraph` monospace (giữ lang trong nhãn nếu cần).
   - `image` (`asset:`/base64) → `ImageRun`.
   - `math` → render fallback (ảnh KaTeX hoặc text) — KaTeX gốc là HTML, DOCX cần xử lý riêng.
2. Áp `FormatPreset` → document styles (font TNR, size 13/14pt, line-height, margin A4, justify).
3. `chapterStartsNewPage` → chèn `PageBreak` trước h1.
4. Cover page + TOC (field TOC của Word hoặc dựng tay từ `formatted.toc`).
5. `docx` `Packer` → buffer `.docx`, tải về. **Không** cần LibreOffice/Pandoc (TechnicalStack §4).

### 5.5 Job lifecycle

1. Tạo `ExportJob` (`status="running"`, `startedAt`).
2. Chạy pipeline target tương ứng.
3. Thành công → `status="done"`, `finishedAt`, trigger download.
4. Lỗi tại stage nào → `status="error"`, ghi `ExportError{ stage, message, recoverable }`, hiện retry.

## 6. Edge Cases & Smart-Operation Notes ("gia vị vận hành")

- **Empty project / empty sections:** Export vẫn ra file (cover + TOC rỗng), không crash; nhắc người dùng nếu rỗng hoàn toàn.
- **Broken image path:** Ảnh `asset:` mồ côi → DOCX/PDF chèn placeholder "ảnh lỗi" thay vì throw; nên chạy Module 3 trước để bắt sớm.
- **Huge table (quá rộng):** PDF dễ tràn lề A4 → áp cờ wide-table từ Module 2 (giảm font / landscape khối đó). DOCX set table `width` + autofit.
- **Deeply nested headings:** DOCX chỉ có HeadingLevel 1–6 → depth > 6 map về level 6 + giữ số đầy đủ trong text.
- **Page-break đẹp:** Giữ caption + hình cùng trang (`break-inside: avoid`); tránh heading mồ côi cuối trang (orphan/widow control trong print CSS).
- **KaTeX trong PDF vs DOCX:** PDF render KaTeX qua Chromium (đẹp). DOCX không có engine math HTML → fallback (ảnh hoặc text) — ghi rõ giới hạn, không vỡ file.
- **Mermaid render failure:** PDF — nếu Chromium không render được diagram (cú pháp sai) → in khối lỗi thay vì trang trắng. HTML — hiện lỗi inline. DOCX — Mermaid là code block → in dạng code + ghi chú "diagram cần xem ở HTML/PDF".
- **Browser print limitations:** Page number/header/footer có thể khác giữa browser. MVP chấp nhận giới hạn này để có PDF dùng sớm; hardening sau mới dùng Puppeteer worker.
- **Puppeteer cold start / timeout (later):** Nếu bật hardening sau, lỗi Chromium phải surface thành `ExportError{ stage:"render-pdf", recoverable:true }`, không treo UI.
- **Large export (nhiều ảnh base64):** HTML self-contained có thể rất nặng → cảnh báo kích thước; PDF/DOCX nén ảnh khi nhúng nếu cần.
- **Font không có (Times New Roman):** Server/Chromium thiếu TNR → fallback serif tương đương + ghi chú; DOCX khai báo font name để Word tự map.
- **Numbering parity:** Ba target PHẢI dùng cùng `formatted` (Module 2) — không tự đánh số lại trong từng exporter (nguồn lỗi lệch số).
- **Còn `error` từ Module 3:** Cho phép export nhưng banner cảnh báo (người dùng tự quyết) — Export không tự chặn cứng.

## 7. Dependencies

### Modules

- **Module 1 (Write):** Nguồn `ReportProject` + `assets` (ảnh để nhúng).
- **Module 2 (Format):** Nguồn `FormattedReport` (AST đánh số + `FormatPreset` + TOC/figures/tables) — đảm bảo 3 target khớp.
- **Module 3 (Check):** Nên chạy trước export; export hiển thị cảnh báo nếu còn `error`.

### Locked stack libs (TechnicalStack.md)

- `unified` + `remark-parse` + `remark-gfm` + `remark-math` + `remark-rehype` — pipeline & intermediate model.
- `rehype-katex` + `katex` — math trong HTML/PDF.
- `rehype-highlight` — code highlight.
- `rehype-stringify` + print CSS nhúng — `report.html`.
- Browser print / print CSS — `report.pdf` MVP.
- `puppeteer` (later hardening / worker) — PDF chính xác hơn sau MVP, chỉ cài khi có contract approve.
- `docx` — `report.docx` (sinh từ mdast, không cần Pandoc/LibreOffice).
- `mermaid` — diagram (client-side trong HTML, Chromium trong PDF).
- `zod` — validate `ExportJob`.
- `Vitest` — test logic merge/parse/map mdast→docx (deterministic).

## 9. Acceptance Criteria

- Exported files include cover page metadata (title/school/course/lecturer/members).
- Exported files preserve heading structure, tables, code blocks, images, và captions.
- Export errors hiển thị rõ (stage + message) và recover được (retry).
- Ba target dùng chung `FormattedReport` (Module 2) → numbering nhất quán HTML = PDF = DOCX.
- PDF MVP dùng browser print / print CSS từ HTML đã format; Puppeteer là hardening sau.
- DOCX sinh trực tiếp từ `docx` lib (không Pandoc/LibreOffice/@react-pdf), là editable version best-effort, không hứa giống PDF 100%.
- Không upload cloud / không gọi dịch vụ ngoài (no cloud Non-goal); PDF MVP chạy qua browser print từ HTML đã format.
