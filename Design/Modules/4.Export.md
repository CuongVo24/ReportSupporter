# Module 4 - Export

## Purpose

Export produces the files users need to submit.

## MVP Responsibilities

- Markdown to HTML.
- Markdown to PDF.
- Markdown to DOCX.
- Report template or README to PDF/DOCX.
- Keep exported content consistent across formats.

## Export Targets

- `report.html`
- `report.pdf`
- `report.docx`

## Export Pipeline Draft

1. Merge project metadata and report sections.
2. Parse Markdown to an intermediate document model.
3. Apply format module transforms.
4. Render to HTML.
5. Render PDF from the formatted HTML.
6. Render DOCX from the intermediate document model or formatted HTML.

## Acceptance Criteria

- Exported files include cover page metadata.
- Exported files preserve heading structure, tables, code blocks, images, and captions.
- Export errors are visible and recoverable.
