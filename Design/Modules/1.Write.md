# Module 1 - Write

## Purpose

Write is the authoring layer of ReportSupporter. It keeps users inside the product by giving them a dedicated report editor instead of a one-shot upload and convert flow.

## MVP Responsibilities

- Markdown editing.
- Live preview.
- Template-generated report skeleton.
- Local draft auto-save.
- Insert image, table, code block, math block, Mermaid diagram, and callout.
- Preserve report metadata such as project title, school, course, lecturer, and members.

## Key Screens

- Workspace editor.
- Template picker.
- Report metadata form.
- Section navigator.
- Asset insert menu.

## Data Model Draft

- `ReportProject`
  - `id`
  - `title`
  - `templateId`
  - `metadata`
  - `sections`
  - `assets`
  - `evidence`
  - `updatedAt`
- `ReportSection`
  - `id`
  - `order`
  - `title`
  - `markdown`
  - `status`

## Acceptance Criteria

- User can create a report from a template.
- User can edit Markdown and see preview changes.
- Draft survives page refresh through local storage or IndexedDB.
- Inserted code blocks keep language metadata for checker/export.
