# Module 1 — Write

> **Vị trí trong stack:** `src/modules/write/` (xem `Design/Modules/Other/TechnicalStack.md` §9).
> **Tuần xây dựng:** W1 (placeholder editor) → W2 (CodeMirror 6 + preview + template + autosave) — theo `Design/RoadMap/MasterRoadMap.md`.
> **Canonical types:** `ReportProject`, `ReportSection` (Contract W1 — `Design/ContractForAI/Core/month1/W1/w1_project_bootstrap_contract.md`).

---

## 1. Purpose

Write là **lớp soạn thảo** (authoring layer) của ReportSupporter và là **màn hình đầu tiên** người dùng nhìn thấy (workspace-first — route `/` PHẢI là editor, không phải landing page, theo TechnicalStack §1).

Mục tiêu giữ chân người dùng: thay vì một flow "upload file → convert → tải về" dùng một lần rồi đi, Write biến sản phẩm thành **nơi người dùng thực sự viết báo cáo**. Người dùng chọn template → app sinh khung báo cáo Markdown → họ viết với preview trực tiếp → draft tự lưu local. Vì toàn bộ Write chạy **client-side** (không login, không cloud — `ProductPRD.md` §6), người dùng có thể mở app và viết ngay, không rào cản đăng nhập.

Triết lý cốt lõi: **Markdown-first**. Người dùng gõ Markdown thật, không phải WYSIWYG ẩn Markdown. Điều này giữ nội dung deterministic để Format/Check/Export ăn chung một nguồn AST (TechnicalStack §3).

---

## 2. Scope

### In scope (MVP — W1/W2)

- Markdown editing qua CodeMirror 6 (`@codemirror/lang-markdown`) — W1 dùng `<textarea>` controlled tạm thời.
- Live preview render từ unified pipeline (remark → rehype → HTML).
- Template-generated report skeleton (sinh `ReportSection[]` từ `TemplateSchema`).
- Report metadata form (title, school, course, lecturer, members).
- Section navigator (điều hướng giữa các `ReportSection`).
- Asset insert menu: image, table, code block, math block (KaTeX), Mermaid diagram, callout.
- Insert image bằng drag-and-drop / clipboard paste (lưu base64 hoặc object URL trong project).
- Local draft auto-save qua IndexedDB (`idb`).

### Out of scope (theo `ProductPRD.md` §6 Non-goals)

- **No login / no cloud sync** — draft chỉ sống local (IndexedDB). Không có account, không multi-device sync.
- **No realtime collaboration** — một người soạn, không có co-editing / presence / CRDT.
- **No AI writing assistant** ở Phase 1 — không auto-generate nội dung, không rewrite. (AI để Phase 3 / W11, sau explicit user action.)
- **No WYSIWYG / contenteditable** (TipTap, Slate, Lexical bị CẤM — TechnicalStack §2). Markdown-first, không rich-text editor.
- **No "convert mọi định dạng"** — Write chỉ nhận Markdown + assets, không import `.docx`/`.pdf` của người khác để chỉnh.

### 3.1 Canonical types (giữ NGUYÊN shape Contract W1 — không đổi)

type ReportProject = {

### 3.2 Module-specific types

```ts
// src/modules/write/types.ts

// Khung báo cáo sinh ra section[]. Lưu trong src/modules/write/templates/.
type TemplateSchema = {
  id: string;                 // "software-project" | "lab-report" | ...
  name: string;               // hiển thị ở template picker
  description: string;
  metadataFields: MetadataFieldSpec[]; // form fields cần điền
  sections: TemplateSectionSeed[];     // khung chương mục
};

type MetadataFieldSpec = {
  key: string;                // "school" | "members" | ...
  label: string;
  type: "text" | "textList";  // textList → string[] (vd: danh sách thành viên)
  required: boolean;
  placeholder?: string;
};

type TemplateSectionSeed = {
  title: string;              // "1. Mở đầu"
  order: number;
  starterMarkdown: string;    // nội dung mẫu / gợi ý
  status: "draft";           // mọi section sinh mới đều "draft"
};

// Asset nhúng trong report (ảnh paste/drop). Lưu trong project record ở IndexedDB.
type ReportAsset = {
  id: string;
  kind: "image";
  fileName: string;
  mimeType: string;          // "image/png" | "image/jpeg" | ...
  data: string;              // base64 data URL (offline-safe, no cloud)
  insertedAt: string;        // ISO 8601
};

// Snippet asset insert menu chèn vào editor.
type SnippetKind = "image" | "table" | "code" | "math" | "mermaid" | "callout";

// Record thực tế lưu IndexedDB (project + assets + draft state).
type StoredProject = ReportProject & {
  assets: ReportAsset[];
  schemaVersion: number;     // để migrate khi đổi shape
};
```

import { z } from "zod";

export const reportSectionSchema = z.object({

  mimeType: z.string(),
  data: z.string(),
  insertedAt: z.string(),
});

export const storedProjectSchema = reportProjectSchema.extend({
  assets: z.array(reportAssetSchema),
  schemaVersion: z.number().int(),
});

## 5. Processing Logic

### 5.1 Pipeline: Create report from template

1. Người dùng mở template picker, chọn một `TemplateSchema`.
2. Render `metadataFields` thành form; validate `required` qua zod khi submit.
3. Sinh `ReportProject`: `id = crypto.randomUUID()`, `templateId = schema.id`, `metadata` từ form.
4. Map mỗi `TemplateSectionSeed` → `ReportSection` (`id` mới, `order`, `title`, `markdown = starterMarkdown`, `status = "draft"`).
5. Set `updatedAt = new Date().toISOString()`.
6. Persist `StoredProject` (kèm `assets: []`, `schemaVersion`) vào IndexedDB.
7. Điều hướng vào workspace editor với section đầu tiên active.

### 5.2 Pipeline: Edit → Preview → Autosave

1. Người dùng gõ trong CodeMirror → `onChange` cập nhật `markdown` của section active (React state).
2. Markdown active → unified pipeline (remark-parse + remark-gfm + remark-math + remark-rehype + rehype-katex + rehype-highlight + rehype-stringify) → HTML preview.
3. Mermaid blocks render **client-side** sau khi HTML mount (Mermaid cần DOM — TechnicalStack §3).
4. Mỗi thay đổi → debounce ~800ms → ghi `StoredProject` vào IndexedDB, cập nhật `updatedAt`, đổi indicator sang "Saved".

### 5.3 Pipeline: Insert asset (snippet)

1. Người dùng chọn `SnippetKind` từ asset menu (hoặc paste/drop ảnh).
2. Với `image`: đọc file → base64 → tạo `ReportAsset`, push vào `project.assets`, chèn Markdown `![alt](asset:<id>)` (preview resolve `asset:` về data URL).
3. Với `table`/`code`/`math`/`mermaid`/`callout`: chèn snippet template tại vị trí con trỏ. Code block snippet **luôn kèm language placeholder** (vd ` ```ts `) để Checker/Export đọc được lang.

### 5.4 Snippet templates (insert)

| Kind | Markdown chèn vào |
| :--- | :--- |
| `image` | `![mô tả ảnh](asset:<id>)` |
| `table` | GFM table 2×2 với header |
| `code` | ` ```ts ` … ` ``` ` (luôn có lang) |
| `math` | `$$ … $$` (remark-math) |
| `mermaid` | ` ```mermaid ` … ` ``` ` |
| `callout` | `> [!NOTE] … ` (GFM blockquote style) |

## 6. Edge Cases & Smart-Operation Notes ("gia vị vận hành")

- **Empty section:** Section có `markdown` rỗng/chỉ whitespace → preview hiện placeholder "Chưa có nội dung", và Checker (Module 3) sẽ flag. Không crash pipeline.
- **Broken / pasted image path:** Ảnh dán nhưng base64 hỏng → preview hiện ô "ảnh lỗi" thay vì vỡ layout. `asset:<id>` không tồn tại trong `assets[]` → render alt text + cảnh báo nhẹ.
- **Huge image paste:** Ảnh > ngưỡng (vd 2MB) → cảnh báo IndexedDB có thể đầy; không tự nén ở MVP nhưng báo trước (xem IndexedDB quota bên dưới).
- **IndexedDB quota exceeded:** Bắt lỗi `QuotaExceededError` khi ghi → hiển thị banner "Bộ nhớ trình duyệt đầy, hãy xóa ảnh lớn / project cũ", **giữ nội dung trong RAM** để người dùng không mất bài đang gõ.
- **Concurrent tabs:** Hai tab cùng project → tab ghi sau thắng (last-write-wins). MVP không khoá; ghi chú rõ là không có realtime (Non-goal).
- **Nested / deep headings trong section:** Người dùng tự gõ `######` (h6) → cho phép; numbering & cảnh báo heading-jump để Module Format/Check xử lý, không chặn ở Write.
- **Mermaid render failure:** Cú pháp Mermaid sai → preview hiện khối lỗi "Mermaid không render được: <message>" thay vì throw làm trắng preview.
- **KaTeX parse failure:** `$$` cú pháp LaTeX sai → rehype-katex render lỗi inline (đỏ) thay vì vỡ preview; KaTeX bật `throwOnError: false`.
- **Markdown cực dài / nhiều section:** Debounce autosave + chỉ re-render preview của section active để giữ mượt; không render toàn bộ project mỗi keystroke.
- **First-load không có project:** Mở app lần đầu (IndexedDB rỗng) → tự mở template picker thay vì màn hình trắng.

## 7. Dependencies

### Modules

- **Module 2 (Format):** Dùng chung pipeline để preview khớp output. Write cung cấp `markdown` + `metadata`; Format chuẩn hoá numbering/TOC/caption.
- **Module 3 (Check):** Đọc AST từ cùng pipeline để báo issue ngay trong workspace.
- **Module 4 (Export):** Tiêu thụ `ReportProject` + `assets` để xuất file.

### Locked stack libs (TechnicalStack.md)

- `CodeMirror 6` (`@codemirror/lang-markdown`, `@codemirror/state`, `@codemirror/view`) — editor (W2+).
- `unified` + `remark-parse` + `remark-gfm` + `remark-math` + `rehype-katex` + `katex` + `rehype-highlight` + `remark-rehype` + `rehype-stringify` — preview pipeline.
- `mermaid` — diagram render (client-side).
- `idb` — IndexedDB draft autosave.
- `zod` — validate metadata / template / dữ liệu đọc từ IndexedDB.
- `Vitest` — test cho logic sinh skeleton & snippet (deterministic).

## 9. Acceptance Criteria

- User can create a report from a template (sinh `ReportSection[]` đúng khung).
- User can edit Markdown and see preview changes (cùng pipeline với export).
- Draft survives page refresh through IndexedDB (`idb`).
- Inserted code blocks keep language metadata for checker/export.
- Inserted images persist trong project (`ReportAsset`) và hiển thị offline.
- First screen is the workspace editor, not a marketing landing page.
- Metadata (title/school/course/lecturer/members) được lưu và dùng lại cho Format/Export cover page.
- Mermaid/KaTeX render failure không làm trắng/vỡ preview.
- No network call required to write/preview (offline-capable).
