## W3 Locked Decisions (review-agreed)

1. **One numbering function, one source of truth.** Heading numbering lives in `src/modules/format/number-headings.ts` and is the *only* path that assigns `1`/`1.1`/`1.1.1`. Preview (W3) and Export (W4) both consume it — preview numbering = PDF = DOCX. No regex numbering, no second counter.

2. **Format reads the W2 AST; it never re-parses.** Format consumes `parseMarkdown` (mdast) from `src/lib/markdown-pipeline.ts`. There is no second parser. Format transforms the AST and emits data (`TocNode[]`, flags); it never renders to a file (that is Module 4).

3. **Checker is AST-first and offline.** Every rule reads mdast/hast nodes from `CheckContext.sectionAsts` **except** `placeholder-text`, which is the only rule allowed to regex raw text. No rule fetches the network — "check link" = "link string exists/parses". Each `section.markdown` is parsed **once** into the context cache, so cost is O(sections), not O(rules × sections).

4. **Rule ids are a public contract and are hard-tested.** `ReportIssue.id` *is* the checker rule id (user-visible). The ids in `3.Check.md` §5.2 are locked; each rule has a Vitest test asserting its id, severity, and trigger. W1's `placeholder-text` + `code-block-no-lang` ids carry over unchanged.

5. **Template-aware, no false positives.** `missing-project-links` / `missing-member-table` / `missing-required-evidence` only fire for software/group templates (via `templateId` + `template.requiredSections` / `requiredEvidenceKinds`). `missing-conclusion` / `missing-references` read `requiredSections` as the primary source, keyword set only as fallback. A lab report never gets a "missing GitHub link" warning.

6. **Format flags, Check reports — Format never rewrites the user's structure.** Heading-level jumps, wide tables, and missing captions are *flagged* by Format and surfaced as issues by Check. Format does not "fix" the heading tree or inject captions.

7. **Canonical shapes are added verbatim; intermediates stay local.** `TocNode`/`CaptionEntry`/`FormattedReport`/`CheckRule`/`CheckContext`/`CheckResult` are copied into `src/types` exactly as in `CanonicalTypes.md` (change the `.md` first if a shape must change). `HeadingNode` (numbering intermediate) is internal to `src/modules/format` — not persisted, not cross-module, so not canonical.

8. **Readiness score is a draft signal, not a gate.** Score reflects issue count/severity only, never content quality; the badge is advisory. Export gating (W4) may *reference* it but W3 ships no pass/fail gate.

## Risks

| Risk | Level | Mitigation |
|---|---:|---|
| Numbering diverges between preview ↔ export | High | Single numbering function on mdast; Export (W4) reuses it (`2.Format.md` Acceptance). |
| Checker raw regex → false positives | Medium | Read AST nodes (`TechnicalStack.md` §3); regex only for `placeholder-text`. |
| Rule logic breaks silently on edit | Medium | Vitest **per rule**, ids/severity hard-asserted (`TechnicalStack.md` §7). |
| Checker calls the network | High | Forbidden in Module Check; "check link" = string check only (`VibeCode.md` §4). |
| Re-parsing per rule (slow on big reports) | Medium | Parse once per section into `CheckContext.sectionAsts`; rules read the cache. |
| Readiness score misread as "perfect" | Low | Mark "draft"; reflects issue count/severity only, not quality. |
| Type drift between `.md` and code | Medium | Add canonical shapes verbatim from `CanonicalTypes.md`; change shape → edit the `.md` first. |
| Any file exceeds 200 lines | Medium | One rule per file; split engine / score / panel / badge (`VibeCode.md` §3). |
