## W2 Locked Decisions (review-agreed)

1. **One pipeline, one source of truth.** The `unified` pipeline in `src/lib/markdown-pipeline.ts` is the *only* MarkdownŌåÆHTML/AST path. Preview (W2), Format & Check (W3), and Export (W4) all consume it ŌĆö no second parser anywhere. This kills "preview ŌēĀ export" drift (`TechnicalStack.md` ┬¦3).

2. **Mermaid is client-only.** Mermaid needs the DOM and must never run during SSR. `MermaidRenderer.tsx` renders via dynamic import / `useEffect` only. KaTeX and `rehype-highlight` run deterministically inside the pipeline (no client dependency for those).

3. **CodeMirror stays controlled, but split for the 200-line rule.** Editor config (`@codemirror/state`/`view`/`lang-markdown`) lives in `editor-setup.ts`, snippet insertion in `insert-snippet.ts`, so `EditorPanel.tsx` stays a thin controlled component. Preview re-parse is debounced; the editor does not re-parse on every keystroke.

4. **Skeleton generator reuses W1, does not re-declare types.** `generate-skeleton.ts` builds on the W1 `createProjectFromTemplate` mapper and the canonical `TemplateSchema`/`ReportSection` shapes. Metadata is validated through the existing W1 zod schemas ŌĆö no new shape, no parallel template type.

5. **Images are local asset references, not embedded blobs in Markdown.** Paste/drop produces a `ReportAsset` (CanonicalTypes ┬¦1) referenced from Markdown; size-limited to keep IndexedDB drafts small. Cloud asset storage stays a Non-goal.

## Risks

| Risk | Level | Mitigation |
|---|---:|---|
| Preview pipeline diverges from future export pipeline | High | One `unified` pipeline in `src/lib`; preview/check/export all consume it. |
| Mermaid breaks SSR (needs DOM) | Medium | Render Mermaid client-only via dynamic import / `useEffect`; never SSR. |
| Controlled CodeMirror causes heavy re-renders | Medium | Debounce state; re-parse preview only when typing pauses. |
| Pasted images bloat IndexedDB drafts | Medium | Store asset reference, enforce size limit; no cloud (Non-goal). |
| Any file exceeds 200 lines | Medium | Split editor-setup / insert-snippet / pipeline into small files (`VibeCode.md` ┬¦3). |
| New deps leak past the W2 matrix | Medium | Matrix locked above; `puppeteer`/`docx`/`qrcode` stay out until their week. |
| Type drift between `.md` and code | Medium | Canonical types in `src/types` match CanonicalTypes; change shape ŌåÆ update the `.md` first. |
