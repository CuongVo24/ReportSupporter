## W1 Locked Decisions (review-agreed)

1. **Evidence types exist in W1.** `EvidenceKind` + `EvidenceItem` (CanonicalTypes §2) are defined under `src/types/` because `ReportProjectBundle.evidence: EvidenceItem[]` and `storedBundleSchema` need them. Evidence Kit *UI* stays W5; in W1 `bundle.evidence = []`.

2. **Template→project mapper + defaults ship in W1.** `[C5]` includes `createProjectFromTemplate` (`id = crypto.randomUUID()`, seeds → sections with `order` + `status:"draft"`, `assets:[]`, `evidence:[]`, `formatSettings = DEFAULT_FORMAT_SETTINGS`, `schemaVersion: SCHEMA_VERSION`). Constants `DEFAULT_TEMPLATE_ID`, `DEFAULT_FORMAT_SETTINGS` (`presetId:"academic-default"`), `SCHEMA_VERSION = 1` live in `src/types`/`src/lib`. This mapper is exactly what `[C5]`'s Vitest verifies (section count, `order`, `status="draft"`).

3. **W1 checker is a text-based subset — NOT the canonical engine.** No `unified`/mdast in W1, so `[C11]` rules run on raw Markdown via a minimal local rule type (`(markdown: string) => ReportIssue[]`), main-thread synchronous. The full `CheckRule`/`CheckContext` (AST, `sectionAsts`) engine arrives in W3. Rule ids still match `3.Check.md` §5.2 (`placeholder-text`, `code-block-no-lang`).

4. **Export stubs return typed results, not throws.** `[C14]` returns `ExportResult { ok: false, error: { stage: "render-pdf" | "render-docx", message: "not implemented until W4", recoverable: false } }`, so W1 call sites already use the W4 shape (zero rework). Visible & recoverable per Coding Standard §6c — nothing swallowed.

## Risks

| Risk | Level | Mitigation |
|---|---:|---|
| Export stack becomes too large for week 1 | Medium | Stub PDF/DOCX in week 1 (return typed error), implement real export in week 4. |
| Editor library choice slows bootstrap | Medium | Use a simple controlled Markdown textarea first, then replace with richer editor. |
| Product scope expands into generic converter | High | Keep MVP acceptance tied to report templates and submission workflow. |
| W1 checker over-built (canonical engine too early) | Medium | Text-based subset only; defer `CheckRule`/`CheckContext` + AST to W3. |
| Type drift between `.md` and code | Medium | Canonical types in `src/types` match CanonicalTypes; change shape → update the `.md` first. |
