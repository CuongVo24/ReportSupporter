## W4 Locked Decisions (review-agreed)

1. **One source, three targets.** HTML, PDF, and DOCX consume the same `FormattedReport`. No exporter re-numbers headings/captions. Numbering is identical HTML = PDF = DOCX because it comes from W3's single numbering function (`4.Export.md` §6 "Numbering parity").

2. **PDF MVP = browser print; Puppeteer is OFF.** The default and only shipped PDF path is `exportPdfViaBrowserPrint()` from the formatted HTML. `renderPdfWithPuppeteer()` stays a disabled stub (no Chromium dep installed) for later hardening. Header/footer/page-number are documented as browser best-effort — MVP makes no pixel-parity promise (`4.Export.md` §5.3).

3. **DOCX from mdast via `docx` only.** No Pandoc, no LibreOffice headless, no `@react-pdf/renderer` (banned — `TechnicalStack.md` §4). DOCX is generated directly from the mdast AST, deterministic, editable best-effort; it does not promise to match the PDF 100%.

4. **Errors are typed, visible, recoverable — never swallowed.** Every exporter returns `ExportResult` and every job tracks `ExportError { stage, message, recoverable }`. The UI shows the stage and a retry; it never hangs (`4.Export.md` §5.5 / Acceptance).

5. **`FormattedReport` assembled in the export front pipeline (W3 reconciliation).** W3 shipped numbering + TOC data only; W4 assembles the full `FormattedReport` (toc/figures/tables/preset/hast/mdast) in `prepare-export.ts` by reusing the W3 Format public functions + the W2 pipeline. No second parser, no second numbering.

6. **HTML is self-contained and offline.** Images embed as base64; Mermaid embeds a client-side render script. No cloud upload, no email, no external fetch (`4.Export.md` §2 Non-goals).

7. **Check does not hard-block Export.** If Module 3 still reports `error`, the export panel shows a warning banner; the user decides. Export never refuses based on checker state (`4.Export.md` §6 "Còn error từ Module 3").

8. **Existing files are replaced/implemented, not re-created.** The raw-`<pre>` HTML stub is fully replaced by pipeline HTML (kills preview≠export drift); PDF/DOCX stubs get real bodies; `src/types/export.ts` is verified against CanonicalTypes §8, not duplicated.

9. **All three exporters are synchronous; the lone async (DOCX zip) is isolated.** `exportHtml` / `exportPdfViaBrowserPrint` / `exportDocx` are synchronous functions — none returns a `Promise`. HTML builds its Blob synchronously; PDF triggers `window.print()` synchronously; `exportDocx` synchronously builds the `docx.Document` (build errors surface synchronously). The only asynchronous operation in the module — `docx`'s `Packer.toBlob` (JSZip has no sync packer) — is extracted into a single `packDocx(doc): Promise<Blob>` helper that `use-export` awaits **once** at download time. The exporter API stays sync and uniform; no swallowed errors.

## Risks

| Risk | Level | Mitigation |
|---|---:|---|
| PDF ≠ preview HTML (numbering/caption drift) | High | PDF renders from the same formatted HTML + W3 numbering; one `FormattedReport` source. |
| "Convert every format" scope creep | High | Only `report.html/pdf/docx` from the user's own Markdown (`ProductPRD.md` §6 Non-goals). |
| Browser print differs across browsers | Medium | Standard A4 print CSS; header/footer/page-number documented as best-effort; Puppeteer deferred. |
| DOCX unfaithful on complex Markdown | Medium | Controlled mdast→docx mapping; Vitest on table/code/image/heading/math fallback samples. |
| Export swallows exceptions | Medium | Typed `ExportResult`/`ExportError`; visible + recoverable + retry (`4.Export.md` Acceptance). |
| `docx` is the only new dep but leaks scope | Low | Exact pin `docx`; `puppeteer`/`@react-pdf`/Pandoc stay out (banned/deferred). |
| HTML stub left in place → preview≠export | Medium | Stub fully replaced by `renderMarkdown`/formatted-hast HTML. |
| Type drift between `.md` and code | Medium | Export consumes canonical `FormattedReport`/`Export*`; module-only `CoverPageData`/`ExportInput` per §3.2. |
