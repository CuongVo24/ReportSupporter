## W7 Locked Decisions (review-agreed)

1. **One numbering source, everywhere.** A single caption registry numbers figures/tables; body, LoF/LoT, PDF, and DOCX all read from it. No second numbering path — preview = PDF = DOCX (`4.Export.md §6`).

2. **Caption normalization is NEW Format logic; the check rule stays a check.** `format/captions.ts` + `caption-registry.ts` do the numbering; `check/rules/captions.ts` keeps only its warn-on-missing-caption role. They are not merged.

3. **LoF/LoT consume existing settings, placed after TOC.** Generators read the W3-shipped `includeListOfFigures` / `includeListOfTables` flags (`DEFAULT_FORMAT_SETTINGS`) — no new settings shape; LoF/LoT render right after the TOC in preview and all exports.

4. **References rules are basic, suggestive, non-strict.** Empty section / malformed-or-empty entry / ordering only, each with a `suggestion`. No APA/IEEE style engine, no citation parser (Out of scope) — avoids false positives.

5. **W7 hardens the real W4 export, not stubs.** `print-css.ts` / `export-pdf.ts` / `mdast-to-docx.ts` already exist and run; W7 tunes page-break, header/footer stability, and DOCX numbering/page-setup parity in place.

6. **PDF MVP stays browser-print; Puppeteer stays OFF.** Page-break and header/footer are print-CSS / browser best-effort. Pixel-perfect rendering is not promised; Puppeteer is only touched under a separate approved hardening contract.

7. **Rules register in the registry; no new export lib.** `referencesRule` is added to `RULES_REGISTRY` (`registry.ts`). No `puppeteer`/`@react-pdf`/Pandoc/LibreOffice — W7 installs nothing.

## Risks

| Risk | Level | Mitigation |
|---|---:|---|
| Caption numbering diverges between LoF/LoT and body | High | One caption registry feeds body + LoF/LoT + both exports (Locked #1). |
| Browser print page-break differs across browsers | Medium | Standard A4 print CSS on multi-template samples; open a Puppeteer hardening contract only if insufficient. |
| References rules too strict → false positives | Medium | Basic rules (empty/malformed/order) with suggestions; no citation-style enforcement (Locked #4). |
| DOCX layout doesn't match PDF | Medium | Same caption/numbering source + page-setup parity in `mdast-to-docx.ts`; checklist + before/after samples. |
| Re-declaring Format settings / re-numbering headings | Low | Settings already in `DEFAULT_FORMAT_SETTINGS` (verify only); captions reuse chapter index from `number-headings.ts`. |
| Scope creep into citation engine or W8 packaging | Low | Citation parser + `evidence.zip`/README generator explicitly out of scope (→W8 / later). |
| Any file exceeds 200 lines | Low | Split captions / registry / LoF-LoT / references / docx-checklist into small files (`VibeCode.md §3`). |
