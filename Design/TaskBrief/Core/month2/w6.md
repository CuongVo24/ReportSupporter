## W6 Locked Decisions (review-agreed)

1. **A template is data, not a code path.** Every report type is a `TemplateSchema`; all flow through the one `generateSkeleton` (W2) and the one checker (W3). Adding a type never adds a generation/validation branch (`1.Write.md`).

2. **The template *type* is canonical and already shipped — populate, never re-declare.** `TemplateSchema` (CanonicalTypes §5) with `requiredSections` / `metadataFields` / `requiredEvidenceKinds` / `requiresToc` exists from W1. W6 fills these fields for new templates; changing the shape means editing `CanonicalTypes.md` first.

3. **The picker is already multi-template — feed it a registry.** `TemplatePicker` renders all `templates[]`. W6 adds `templates/index.ts` (`ALL_TEMPLATES` + `getTemplate`) and wires it in; no picker rewrite.

4. **README-to-report is deterministic mapping, not AI.** `importReadme` maps README mdast headings into a fixed report skeleton — offline, no model, no network, no invented prose (AI content is a W11 optional, behind its own contract).

5. **Reusable section blocks are shared, not copied.** Member-responsibility and project-timeline live in `src/modules/write/sections/` and are embedded by reference into templates — no per-template duplication.

6. **The checker already reads required sections — W6 verifies, not rewrites.** `missing-sections.ts` resolves the active `TemplateSchema` and gates on its `requiredSections`/`requiresToc`. W6 only extends test coverage per new template.

7. **One public surface.** Everything ships from `@/modules/write` (`index.ts`) — `ALL_TEMPLATES`, `getTemplate`, the section builders, `importReadme`. Other modules never deep-import a template file.

## Risks

| Risk | Level | Mitigation |
|---|---:|---|
| Templates drift into "convert every input format" | High | Only README/Markdown input; each template is a fixed report skeleton (`ProductPRD.md §6`). |
| Checker logic branches per-template → debt | Medium | Templates declare `requiredSections` as data; checker reads data, no hard-coded per-template `if`. |
| README import maps the wrong sections | Medium | Controlled mdast heading mapping + Vitest on README samples; unknown content into a controlled bucket. |
| Re-declaring the canonical `TemplateSchema` | Low | Reuse `src/types/template.ts`; Day-1 "new fields" is a no-op verification (reconciliation). |
| Section blocks duplicated across templates | Low | Extract member/timeline into `src/modules/write/sections/` and embed by reference. |
| README import or template file exceeds 200 lines | Low | Split per template file + separate `readme-import.ts` + per-section builders (`VibeCode.md §3`). |
| Picker rewritten unnecessarily | Low | Picker is already multi-template; only add the registry behind it (reconciliation). |
